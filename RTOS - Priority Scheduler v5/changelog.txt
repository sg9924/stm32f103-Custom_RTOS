1: INITIAL
    - Introduce PendSV Exception Handler
    - PendSV Handler manages the context switch and the scheduler function call. This is a naked function.
    - Systick Handler manages the Tick increment and the task unblocking. This is not a naked function unlike the previous implementation.
    - This implementation provides the correct response to a task yield call.
        - In the previous implementation where we were using only the systick handler to manage everything: context switch, scheduler call, systick increment and the task unblocking.
        - invoking the task yield artificially accelerates the tick, as we are intentionally invoking the systick exception instead of letting it happen according to the configuration whether its 1 ms or 10 ms...
    - added a test function which blinks 3 LEDs connected to specific GPIOs

2: TASK
    - Added return in taskAdd_Priority to return the TCB of the task being added which can be used later in the main program

3: BUG FIX
    - Bug: incorrect size calculation in Startup code where we initialize .bss and copy .data to SRAM

4: KERNEL, TASK
    - Added support for individual unique stack size for each task
    - common stack size is also present
    - config is provided to switch between the 2 options
    - Task
        - taskAdd APIs have a new parameter - stack_size_word, this argument will be ignored when the RTOS is configured to use the common stack size
    - Kernel
        - New API: Stack_Allocate(uint32_t size_in_words), used to allocate the stack for each task, basically a arena allocator but not actually dynamic memory allocation
          just yet. Later the first heap of Arena Allocator type will be added and this code will be modified to use that
        - rtosKernel_TaskStackInit has been modified to handle the Stack Init for both stack configuration options
        - minor modification in assert API

5: Bug Fixes - KERNEL, TASK
        - Idle Task Selection Condition Logic in the Scheduler
        - Task Unblock: the block tick condition doesn't account for the tick overflow
